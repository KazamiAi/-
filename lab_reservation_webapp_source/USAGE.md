# 实验室预约系统使用说明

## 问题描述
每次关闭应用窗口后，再点击网址进入时都会显示服务不可用。

## 原因分析
当您关闭应用窗口时，Node.js服务器进程也会随之停止，导致再次访问网址时无法连接到服务器。

## 解决方案
要解决这个问题，您需要确保服务器在后台持久化运行。以下是几种方法：

### 方法一：使用终端手动启动（推荐）

1. **打开命令提示符（CMD）**
   - 按下 `Win + R` 键，输入 `cmd` 并按回车键

2. **进入项目目录**
   ```bash
   cd "d:\学习资料\系统\lab_reservation_webapp"
   ```

3. **启动服务器**
   ```bash
   node backend/server.js
   ```

4. **保持窗口打开**
   - 不要关闭这个命令提示符窗口
   - 服务器将持续运行，即使您关闭了浏览器窗口

### 方法二：使用启动脚本

我们已经为您创建了简单的启动和停止脚本：

1. **启动服务器**
   - 双击运行 `start_server.bat` 文件
   - 这将打开一个新窗口并启动服务器
   - 不要关闭这个窗口

2. **停止服务器**
   - 双击运行 `stop_server.bat` 文件
   - 这将停止所有Node.js服务器进程

### 方法三：使用PM2持久化运行（高级）

如果您希望服务器在系统启动时自动运行，可以使用PM2：

1. **安装PM2**
   ```bash
   npm install pm2 -g
   ```

2. **启动服务器**
   ```bash
   pm2 start backend/server.js --name lab-reservation
   ```

3. **设置开机自启**
   ```bash
   pm2 startup
   pm2 save
   ```

4. **停止服务器**
   ```bash
   pm2 stop lab-reservation
   ```

## 验证服务器状态

您可以通过以下方式验证服务器是否正在运行：

1. **访问测试端点**
   - 打开浏览器，访问 `http://localhost:3000/test`
   - 如果看到 `{"message":"服务器运行正常","timestamp":"2023-12-10T12:34:56.789Z"}` 则表示服务器正在运行

2. **检查进程**
   - 打开任务管理器，切换到 "进程" 标签
   - 查找名为 `node.exe` 的进程，如果存在则表示服务器正在运行

## 访问应用

服务器启动成功后，您可以通过以下地址访问应用：
- 主页面：`http://localhost:3000`
- 登录页面：`http://localhost:3000/login.html`
- 个人信息页面：`http://localhost:3000/profile.html`

## 常见问题

### Q: 为什么访问网址时显示 "服务不可用"？
A: 可能是服务器没有运行。请按照上述方法重新启动服务器。

### Q: 如何停止服务器？
A: 您可以：
   - 关闭启动服务器的命令提示符窗口
   - 运行 `stop_server.bat` 脚本
   - 在任务管理器中结束 `node.exe` 进程

### Q: 服务器启动后，为什么浏览器无法访问？
A: 请检查：
   - 服务器是否正在运行
   - 端口 3000 是否被其他程序占用
   - 防火墙是否阻止了连接
